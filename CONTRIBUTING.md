# Contributing

Contributions are welcome — especially updates to the reference files, which need to stay current as the AI hiring landscape changes fast.

---

## What's Worth Contributing

**Reference file updates** (highest value)
- New research that confirms, contradicts, or updates a claim in `ai-screening-research.md`
- New ATS platform behavior you've verified
- Salary data source updates
- Regulatory changes (new laws, enforcement actions, case outcomes)

**Skill improvements**
- Better prompting patterns that produce more accurate output
- Missing report sections or evaluation dimensions
- New company types or role archetypes that aren't handled well

**Bug reports**
- The skill misidentified files, skipped a section, or produced garbled output
- A specific ATS or platform is described inaccurately
- A research source is dead or paywalled

---

## How to Submit

1. Fork the repo
2. Make your changes
3. Update `CHANGELOG.md` with what you changed and why — include the source if it's a factual correction
4. Open a pull request with a clear description

For reference file changes, include the source you're drawing from (publication, date, URL if available). Unsourced factual claims won't be merged.

---

## Reference File Review Cadence

The AI hiring landscape moves fast. The reference files in `resume-review/references/` and `company-intel/references/` should be re-validated against current sources every 6 months. If you spot something outdated, open an issue or submit a PR — even a small correction helps.

---

## What Not to Submit

- Changes that add editorial opinion to the Political Disposition section — it's intentionally neutral and fact-only
- Reframe strategies or scoring changes that encourage candidates to misrepresent experience
- Anything that would make the skills give worse advice to help people "game" systems in ways that backfire
